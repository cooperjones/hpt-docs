# Modal Notification Manager Component
<br/>

## ModalNotificationManager

The ModalNotificationManager is automatically included in the ActionModal, FlexModal, and FullscreenModal components, so it generally will not need to be called directly at implementation. It uses a provider component which exposes a function which can be used to push a notification. The corresponding consumer component should be used to access the sendNotification function where necessary. The function requires a single object argument to pass the notification data to the manager.

## Modal Notification Context Consumer

Use the consumer component to access and use the send notification function. See below for function shape.

**Note:** The consumer should only be used within the context of an ActionModal, FlexModal, or FullscreenModal component.


### Usage (React)

<Source code={`
import {ModalNotificationConsumer} from '@highpoint/reason-ui-elements';\n
<ModalNotificationConsumer>
  {
     (sendNotification) =>
      <Button onClick={() => sendNotification({

           variant: "success"
         }
)}>
		Make Notification
</Button>
  }
</ModalNotificationConsumer>
`} />

## sendNotification Function

```({
({
message: string, 
variant: string ['success' | 'failure']
}) => unit
```

The sendNotification function which is passed from the provider in NotificationManager accepts a single object as an argument. It should contain some or all of the following fields:

1. message: reactElement - The body message for the notification
2. variant: string - The type of notification (either success, failure, or standard)
