# Notification Manager Component
<br/>

## NotificationManager

The NotificationManager should wrap around the entire app. It uses a provider component which exposes a function used to push a notification. The corresponding consumer component should be used to access the notification function where necessary. The function requires a single object argument to pass the notification data to the manager.

### Usage (React)

<Source code={`
import {NotificationManager} from '@highpoint/reason-ui-elements';\n
<NotificationManager> 
    <App /> 
</NotificationManager>
`} />

### Usage (Reason)

<Source code={`
<HighpointReasonUiElements.NotificationManager> 
    <App /> 
</HighpointReasonUiElements.NotificationManager>
`} />

## Notification Context Consumer

Use the consumer component to access and use the send notification function. See below for function shape.

### Usage (React)

<Source code={`
import {NotificationConsumer} from '@highpoint/reason-ui-elements';\n

<NotificationConsumer>
 {
    (sendNotification) =>
     <Button onClick={() => sendNotification({
           icon: <MyIconComponent/>,
           title: "Notification Title",
           message: "This is a notification message",
           confirmText: "Got it",
           cancelText: "Cancel"
        }
     )}>
       Make Notification
     </Button>
 }
</NotificationConsumer>
`} />

### Usage (Reason)

<Source code={`
<HighpointReasonUiElements.NotificationManager.Consumer>
...{
     sendNotification => 
     <Button onClick={_e => sendNotification(HighpointReasonUiElements.NotificationManager.notification(
        ~title="Notification Title",
        ~icon=<MyIconComponent />,
        ~message="Notification message",
        ())
      )}>
       "Send Notification"
     </Button>
}
</HighpointReasonUiElements.NotificationManager.Consumer>
`} />


## sendNotification Function

```({
title: string, 
icon: reactElement, 
message: string, 
cancelText: string, 
confirmText: string
}) => unit
```

The sendNotification function which is passed from the provider in NotificationManager accepts a single object as an argument. It should contain some or all of the following fields:

1. title: string - The title text for the notification
2. icon: reactElement - The icon for the notification
3. message: reactElement - The body message for the notification
4. confirmText: string - The text for the confirm button on the notification. If not passed, no button will be rendered.
5. cancelText: string - The text for the cancel button on the notification. If not passed, no button will be rendered.
